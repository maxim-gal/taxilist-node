# AGENTS.md

## Обзор проекта
Сайт таксопарков на Express + EJS + SQLite (sql.js) с админ-панелью. Русская локаль.

## Команды
- **Запуск**: `npm start` (запускает `node app.js`)
- **Порт**: По умолчанию 3000, настраивается через `PORT` в .env
- **Нет lint, test, typecheck** — не настроены

## Окружение (.env)
```
PORT=3000
ADMIN_PASSWORD=admin123        # Пароль для входа в админку
SESSION_SECRET=change_this...  # Сменить в продакшене
```

## Архитектура
- `app.js` — Точка входа, инициализация БД, Express, сессии, статика, роуты
- `db.js` — Обёртка SQLite через sql.js (WASM), сохраняет в `database.sqlite`
  - Таблицы: `taxis`, `articles`, `settings` (синглтон, id=1)
  - Авто-миграция `settings` (добавляет `about_image`, `contacts_image`)
  - Сидит начальные данные при первом запуске
  - Экспортирует `getAsync`, `allAsync`, `runAsync` в `req.db`
- `routes/index.js` — Публичные роуты: главная, такси, о проекте, контакты, статьи
- `routes/admin.js` — Админ-роуты (защищены сессией)
  - Логин: `/admin/login` (проверяет `ADMIN_PASSWORD`)
  - CRUD для такси, статей, настроек
- `views/pages/*.ejs` — Шаблоны страниц
- `views/partials/*.ejs` — Части (head, footer)
- `public/` — Статические файлы

## Ключевые соглашения
- Методы БД подключены к `req.db` через middleware (`app.js:27-30`)
- Авторизация админа через `req.session.isAdmin` (устанавливается при логине)
- Все админ-роуты под `/admin` требуют логин (`admin.js:25`)
- Таблица `settings` — синглтон (`id=1` с CHECK constraint)
- SQL использует параметризованные запросы (`?` плейсхолдеры)

## Подводные камни
- `sql.js` — WASM, первая загрузка асинхронная, нет нативных зависимостей
- БД авто-сохраняется при каждом `runAsync` (`db.js:145`)
- Нет системы миграций кроме добавления колонок в `settings`
- Нет валидации/санитизации входных данных в админке
- Секрет сессии по умолчанию `'default_secret'` если не задан
